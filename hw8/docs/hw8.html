Problem 1: Willy Wazoo's Boolean Expressions [70 points]

Willy Wazoo has written a program that parses, evaluates and prints boolean expressions, and his code works! The parser takes a string expression in preorder form and constructs a boolean expression tree (the one we did in class). In preorder, the operator (AND, OR) appears before the two operands of the expression; e.g., x AND y is AND x y in preorder, and x AND y OR z is OR AND x y z in preorder. We restrict AND and OR to two operands. The example from lecture, (x OR true) AND y translates to AND OR x true y in preorder. Note that in preorder one need not worry about order of operations. The reason why Willy chose preorder, is that it is much easier to parse an expression if it is in preorder form, than if it is in the "normal" inorder form.

We have added Willy's code in src/hw8/ and his test cases in src/hw8/test. Willy's code parses, evaluates in a context of variable-to-boolean mappings, and prints the expressions. It prints the expressions in preorder and in inorder. Printing in inorder is a little bit more involved because one must take into account order of operations and insert parentheses when needed. In boolean arithmetic NOT has highest precedence, followed by AND, followed by OR. If the preorder expression is AND OR x y z, this should be printed as (x OR y) AND z in inorder, to account for the fact that operand y associates with the OR. x OR y AND z is equivalent to x OR (y AND z), OR x AND y z in preorder.

Willy's code works fine, except that he forgot to implement the NOT expression! He tried to add a NOT, but quickly became frustrated beause it required too many changes throughout the code.

Your task is to fix Willy's code. You must identify all code smells, refactor and implement NOT and still pass all Willy's tests. List all code smells you identify in hw8/answers/problem1.txt. List all refactorings you use in hw8/answers/problem1.txt as well.
